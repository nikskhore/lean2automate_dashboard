# Deployment Guide — Finance Tracker

Target architecture (always-on):

```
Web (React build)  ->  Cloudflare Pages        https://app.<domain>
API (Express/Node) ->  Render (Node service)   https://api.<domain>
Database           ->  Supabase Postgres       (already set up)
File storage       ->  Supabase Storage OR Cloudflare R2 (S3-compatible)
Recurring cron     ->  Cloudflare Cron Trigger -> POST /api/internal/run-recurring
```

Secrets never live in git — `.env` files are gitignored. All secrets are set in the host dashboards.

---

## 1. Push code to GitHub

Create an **empty** private repo (no README) at github.com, then locally:

```bash
cd c:/Lean2Automate/Lean2Automate_Dashboard
git add -A
git commit -m "Finance Tracker: Phase 1 + 2"
git branch -M main
git remote add origin https://github.com/<you>/finance-tracker.git
git push -u origin main
```

## 2. File storage (S3-compatible)

Pick one. Both use the same `S3_*` env vars on the API.

**Option A — Supabase Storage (fewest accounts, no card):**
1. Supabase dashboard → **Storage** → **New bucket** → name `receipts` (Private).
2. **Storage → S3 Connection** (or Settings) → generate **Access Key ID** + **Secret**.
3. Values:
   - `S3_ENDPOINT=https://<ref>.storage.supabase.co/storage/v1/s3`
   - `S3_REGION=<your project region>` (e.g. `ap-northeast-1`)
   - `S3_BUCKET=receipts`

**Option B — Cloudflare R2:**
1. Cloudflare dashboard → **R2** → create bucket `receipts` (enabling R2 needs a card on file).
2. **R2 → Manage API Tokens** → create token → get Access Key ID + Secret + the S3 endpoint.
3. Values:
   - `S3_ENDPOINT=https://<accountid>.r2.cloudflarestorage.com`
   - `S3_REGION=auto`
   - `S3_BUCKET=receipts`

## 3. Deploy the API to Render

1. render.com → sign in with GitHub → **New + → Blueprint** → pick this repo (it reads `render.yaml`).
2. Set the secret env vars in the dashboard:
   - `DATABASE_URL` = your Supabase Session-pooler URI
   - `CORS_ORIGIN` = `https://app.<domain>` (the web origin)
   - `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY` (from step 2)
   - (`JWT_SECRET` and `CRON_SECRET` are auto-generated by the blueprint)
3. Deploy. Health check: `https://<service>.onrender.com/api/health`.

> Free plan sleeps after 15 min idle (first request cold-starts ~30–60s). ~$7/mo keeps it always warm.

## 4. Deploy the web to Cloudflare Pages

1. Cloudflare → **Workers & Pages → Create → Pages → Connect to Git** → pick this repo.
2. Build settings:
   - **Root directory:** `web`
   - **Build command:** `npm install && npm run build`
   - **Output directory:** `dist`
3. Environment variable: `VITE_API_URL = https://api.<domain>/api`
4. Deploy → you get `https://<project>.pages.dev`.

## 5. Custom domain (your Cloudflare domain)

- Pages project → **Custom domains** → add `app.<domain>`.
- Render service → **Settings → Custom Domain** → add `api.<domain>` → Cloudflare DNS: add the CNAME Render shows (set to **DNS only** / grey cloud so Render's TLS works, or follow Render's proxied instructions).
- Update `CORS_ORIGIN` (Render) and `VITE_API_URL` (Pages) to the final domains and redeploy.

## 6. Recurring cron (Cloudflare)

Create a Cloudflare **Cron Trigger** (a tiny Worker) that runs daily and calls:

```
POST https://api.<domain>/api/internal/run-recurring
Header: X-Cron-Secret: <CRON_SECRET from Render>
```

(Until then, "Run due now" on the Recurring screen works manually.)

## 7. Verify live

- `GET https://api.<domain>/api/health` → `{status:"ok"}`
- Open `https://app.<domain>`, register a new account, add a transaction with a receipt (confirm it lands in the bucket and downloads back), run a P&L export.
